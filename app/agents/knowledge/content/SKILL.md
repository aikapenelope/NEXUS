# Content Creation Skill

You are producing publication-ready content. Follow these standards for
consistency and quality across all content types.

## Content Structure

### Blog Posts / Articles
1. **Hook** (first 2 sentences): State the problem or insight clearly
2. **Context** (1 paragraph): Why this matters now
3. **Body** (3-5 sections): One idea per section, with examples
4. **Conclusion**: Actionable takeaway, not a summary
5. **Meta**: Suggested title, meta description (155 chars), tags

### Social Media (Twitter/X)
- Lead with the insight, not the setup
- One thread = one idea, max 5 tweets
- First tweet must stand alone
- Include a hook that creates curiosity
- End with a question or call to action

### Technical Documentation
- Start with "what" before "how"
- Include a working code example within the first 3 paragraphs
- Use progressive disclosure: simple case first, edge cases later
- Every code block must be copy-pasteable and runnable

## Writing Principles

### Clarity
- One idea per paragraph (3-4 sentences max)
- Active voice by default
- Concrete nouns over abstract ones
- "Use X to do Y" over "X can be used to do Y"

### Tone
- Professional but not corporate
- Direct but not aggressive
- Confident but not arrogant
- Technical accuracy over simplicity (don't dumb down)

### Formatting
- Headers create scannable structure
- Bold for key terms on first use
- Bullet points for lists of 3+ items
- Code blocks for anything technical

## Quality Checklist

Before delivering content:
- [ ] Does the first sentence hook the reader?
- [ ] Is every paragraph necessary?
- [ ] Are claims supported with evidence or examples?
- [ ] Is the tone consistent throughout?
- [ ] Would the target audience understand every term used?
- [ ] Is there a clear call to action or takeaway?
- [ ] Word count matches the requested format?
